# 럭키 시뮬레이터 로드맵

> 북극성: **실제 리스 시뮬봇 카드 한 장이 300턴 동안 상태 불일치 없이 작동하는가?** (디싱크 0)
> 정체성·설계 근거는 [ADR 0002](./adr/0002-lucky-simulator-risu-compatible-player.md).

## 전략: 카나리아 사다리

리스 시뮬봇의 3가지 인코딩 스타일을 대표 카드 하나씩으로 정복한다.

| 카나리아 | 스타일 | 증명하는 것 |
|---|---|---|
| ① 용사여관 | Lua 태그 파서 | 태그→이벤트 번역 + 엔진 소유 상태 (완전 시뮬 모드) |
| ② DOMINIUM | CBS/regex | 앱 코드 수정 0으로 두 번째 카드 = 범용성 |
| ③ Tesselia | 순수 로어북 | 모듈 0개 일반 채팅 모드에서도 롤백·저장 = 3모드 완성 |

## 단계

- **Phase 0 — 엔진 기반 (완료)**: 결정론 커널·모듈·세션 저널·근거 기억, pnpm 모노레포 마이그레이션 + 보안 척추, ADR 0002.
- **Slice 1 — 용사여관 관통 (진행중)**: ✅ 태그 번역기(직역/근사/보존) ✅ 디싱크 하버스 300턴 ✅ 사실 영수증 + 되돌리기 UI / ⬜ 카드 임포트 ⬜ 호환성 여권 UI ⬜ 번역기 실세션 연결.
- **Slice 2 — DOMINIUM**: CBS/regex 계약. 성공 기준: 앱 코드를 고치지 않고 카드 선언만으로 작동(번역 계층의 범용화).
- **Slice 3 — Tesselia**: 일반 채팅 모드(모듈 0). 시뮬이 아닌 카드도 롤백·체크포인트·저장을 얻는다(보편 USP).
- **Slice 4 — 이사 환경**: 프롬프트 프리셋 임포트, 페르소나, 로어북, 두 줄 레일 검사기(읽기전용). "동일한 환경" 약속의 이행.
- **Slice 5 — 장기 플레이 품질**: 체크포인트 브랜칭, 세션 내보내기/공유, 장기기억 벤치 강화, 성능.
- **Phase 3 — 커뮤니티 베타**: 카드별 호환성 여권 DB, "럭키에서 돌아감" 배지.

## 원칙 (불변)

1. **디싱크 0 최우선** — 기능보다 무결성. 카나리아마다 하버스 먼저.
2. **카드 Lua는 실행하지 않는다** — 의미를 번역한다. 못 옮기면 실행 말고 보존.
3. **보안 척추 불가침** — BUTTON_ONLY_EVENTS, sessionIntegrity, maskSecrets, support.own(), Resilient isCorruption.
4. **각 슬라이스는 관통 단면** — 엔진→UI까지 세로로 뚫고, 매번 독립 감사.

## 리스크

- **LLM 태그 준수율**: 모델이 태그를 빼먹을 수 있음 → 영수증에 "태그 없음"이 보이므로 감지 가능, 재생성으로 복구.
- **CBS/regex 범위**: 전체 재현은 늪 → 카나리아가 실제로 쓰는 부분집합만, 여권에 정직하게 표기.
- **카드 생태계**: 카드 원본은 사용자 로컬에만. 우리는 실행 환경만 제공한다.

## 미해결 목록 (2026-07-12 오너 지적 · 착수 대기)

### 마이그레이션 회귀 (되살려야 할 우리 자산)
- **LLM 임포트 컴파일러 — 증발함.** 커밋 `03074d8`(레거시 앱 제거)이 `compilerPrompt.js`(233)·`schemaPatch.js`(215)·`cardDiagnosis.js`(193)·`luaMine.js`(488)와 테스트를 함께 지웠고 새 모노레포에 재구현되지 않았다. 현재 `packages/compiler`의 `compileCardDraft`는 **LLM을 쓰지 않는 휴리스틱**(모듈 후보 제안)이고 제작 편집기에만 연결돼 있다. 그 결과 카드 임포트는 `card-project.ts`에서 **ysp 태그가 보이면 여관 스키마를 하드코딩**한다 → 두 번째 카나리아(DOMINIUM)부터 성립 불가.
  - **복원 위치: Slice 8(CBS/DOMINIUM)의 전제.** 카드별 스키마를 뽑아내는 것이 곧 "앱 코드 수정 0으로 두 번째 카드 돌리기"의 방법이다.
  - 참고: 옛 구현은 `git show 03074d8^:app/src/llm/compilerPrompt.js` 등으로 열람 가능.
- (해소됨) Vertex AI 프로바이더 — 같은 커밋이 삭제, `94ec709`로 복원.
- **프로바이더 외 나머지 계층의 마이그레이션 패리티 전수 점검이 아직 안 됐다.** 감사 때 엔진·세션·UI만 봤다.

### 리스 문법 불일치 (UI)
- **첫 화면이 임포트 화면이면 안 된다.** 리스는 열면 채팅 화면(마지막 캐릭터)이고 임포트는 메뉴 안 동작이다. 우리는 카드가 없을 때 임포트 화면을 첫 얼굴로 띄운다 → 임포트는 "다음 화면"으로 내리고, 첫 화면은 채팅(또는 캐릭터 목록)으로.
- **새 채팅 순서가 반대다.** `ChatStore.create`가 `chats.push()`라 새 채팅이 아래로 붙는다. 리스는 최신이 **맨 위**(unshift), 낮은 숫자(1회차)가 아래다.

# 럭키 시뮬레이터 로드맵

> 북극성: **실제 리스 시뮬봇 카드 한 장이 300턴 동안 상태 불일치 없이 작동하는가?** (디싱크 0)
> 정체성·설계 근거는 [ADR 0002](./adr/0002-lucky-simulator-risu-compatible-player.md).
> 상세 작업 순서와 세션 메모는 [BACKLOG](./BACKLOG.md)가 기준.

## 전략: 카나리아 사다리 (2026-07-17 개정 — 3카드 집중)

"모든 카드 자동 실행"이 아니라 **인증 카드를 하나씩 추가하며 범용 모듈을 검증**한다.
현재 최우선 카나리아는 소녀전선:잔불이며, DOMINIUM·Tesselia는 사다리의 다음 칸으로 유지된다.

| 카나리아 | 스타일 | 증명하는 것 | 상태 |
|---|---|---|---|
| 용사여관 | Lua 태그 파서 | 태그→이벤트 번역 + 엔진 소유 상태 | 관통 완료(회귀 고정) |
| **소녀전선:잔불** | PNG+Lua 대형 시뮬 | 네이티브 변환·세 기둥(전투·경영·연애) 완성도 | **진행 중 — 계층 1** |
| DOMINIUM | CBS/regex | 앱 코드 수정 0의 범용성 | 대기(계층 4) |
| Tesselia | 순수 로어북 | 모듈 0 일반 채팅의 롤백·저장 | 대기(계층 4) |

## 현재 체크포인트 (2026-07-20)

- **세이브 에폭 가동** — 업데이트가 저장을 격리하지 않고 봉인·이주한다(v0.1 원장 하위호환,
  integrity 실패만 격리). "업데이트=초기화" 문제 종결.
- GFL 네이티브: 다단계 작전(정찰/교전/돌발/미스터리/보스, 원본 EV_GUIDE 회수)·루팅·조우/보스 영입·
  군수지원·야간전·서약·도감·습격·해체·지휘관 레벨·진형 6칸·병과 역할·세력 상성·관계 11티어
  사다리·1:1 대화·기지 근무·불만도 경제·가공·시설 특화·암시장까지 배포됨.
- 프롬프트 매크로 경고 945→3(안전 부분집합·정적 소거·중복 억제), 300+사건 자동 하버스가
  결정론·프롬프트 평탄성·에폭 왕복을 기계 증명.
- 검증 규율: `pnpm check` + 전체 e2e + 실카드 canary:gfl + (해당 시) 몬테카를로가 커밋 관문.
  CI에는 아직 e2e가 없다 — 계층 4 과제.
- 장기 회차 성능 수술: 채팅 창문 렌더·에셋 수명 상한·화면 revision 분리·구형 저장 압축을 배포했다.
  1,000사건 합성 회차의 로컬 엔진 p95는 2.7ms이며, 30일 GFL 하버스는 607사건·디싱크 0·동일 해시를
  재확인했다. 따라서 세션 전체 Worker 이전과 저장 엔진 교체는 즉시 착수하지 않고 아래 플랫폼 관문을 따른다.

## 공용 플랫폼 준비 — 다봇·계정 동기화·Worker/저장소 관문

소녀전선뿐 아니라 판타지봇·Alternate Hunters V2 등 여러 대형 봇을 같은 플레이어에서 지원할 수 있도록
화면과 세션 사이의 공용 경계를 먼저 만든다. 이 작업은 플레이 수치나 세이브 의미를 바꾸지 않는 준비 공사다.

### 지금 진행하는 순서

1. **구조 감사** — 이미 존재하는 카드 Runtime Worker, SQLite/OPFS 저장 Worker, 행동 WAL, 화면 행동 통로를
   기준으로 중복 구현을 막는다.
2. **공용 행동 계약** — `행동 요청 → 엔진 처리 → 내구 영수증 → 화면 결과`를 한 계약으로 묶고 행동 순번과
   세션 식별자를 둔다. 기존 버튼의 결과·RNG·저널은 바꾸지 않는다.
3. **통합 화면 묶음** — 화면이 raw state를 직접 훑는 대신 행동 후 필요한 selector 결과를 한 번에 받는 경계를
   만든다. 처음에는 전체 diff보다 안전한 화면 단위 묶음을 우선한다.
4. **다봇 어댑터 검증** — 소녀전선 전용 UI를 복사하지 않고 모듈 event/selector/화면 선언으로 두 번째 대형
   카나리아를 연결한다.
5. **동기화 준비 계약** — Firebase 로그인 도입 전에도 세션 ID·행동 순번·부모 상태 해시·결과 해시를 보존해
   모바일과 PC의 동시 진행을 덮어쓰지 않고 분기로 보존할 수 있게 한다. 실제 클라우드 전송은 별도 승인 작업이다.
6. **재계측 후 결정** — 두 번째 대형 봇과 장기 회차에서 아래 관문을 통과하지 못할 때만 전면 Worker화나
   저장 엔진 경쟁 시험을 승격한다.

### 세션 전체 Worker 관문

현재는 **보류**한다. Worker는 화면 멈춤을 격리하지만 같은 기기의 CPU 일을 없애지 않으며, 큰 상태를 매번
복사하면 오히려 발열과 지연이 늘 수 있다. 다음 중 하나가 실기기에서 반복될 때 착수한다.

- 엔진·기억·체크포인트의 메인 스레드 작업이 50ms를 넘는다.
- 최적화 후에도 로컬 엔진 버튼의 결과 표시 p95가 100ms를 넘는다.
- 두 개 이상의 대형 카나리아에서 같은 정지가 재현된다.
- Firebase 증분 동기화 준비가 화면을 막으며, 화면에는 작은 상태 묶음만 보낼 수 있는 계약이 완성됐다.

착수 시에는 `통신 계약 시제품 → 신구 시간 비교 → 세션 정본 전체 이전` 순서를 지킨다. 엔진·RNG·기억·
체크포인트·WAL·undo/redo는 한 정본으로 이동하고, 행동 번호·늦은 응답 폐기·Worker 재시작 후 WAL 복구를
필수 계약으로 둔다. PlaySession 코어는 Worker 밖에서도 기존 결정론 테스트를 그대로 실행할 수 있어야 한다.

### 저장 엔진 교체 관문

현재 브라우저 정본은 공식 SQLite WASM+OPFS Worker이며 실패 시 IndexedDB로 후퇴한다. 봇 수가 늘거나
Firebase를 붙인다는 이유만으로 교체하지 않는다. 다음 조건을 모두 충족할 때만 후보를 경쟁시킨다.

1. 불필요한 전체 복사·직렬화·렌더 비용을 먼저 제거했다.
2. 실제 장기 세션에서 저장 단계가 병목임을 구간 계측으로 확인했다.
3. 현행 OPFS, IndexedDB, 대안 VFS를 같은 기기·같은 데이터·같은 행동 리듬으로 비교했다.
4. 승자가 PC·Android·iPad에서 안전한 폴백, 기존 세이브 무손실 이주, 되돌리기 경로를 제공한다.
5. 일반 행동 저장 p95 100ms 이하, 강제 종료 후 마지막 내구 행동 복구, 결정론·분기·undo/redo 불변을
   모두 통과한다.

Firebase가 도입돼도 로컬 저장은 오프라인 정본으로 남고, 클라우드는 뒤에서 행동 저널과 복구 스냅샷을
동기화한다. 네트워크 응답을 기다려야 버튼 결과가 보이는 구조는 금지한다.

## 원칙 (불변)

1. **디싱크 0 최우선** — 기능보다 무결성. 카나리아마다 하버스 먼저.
2. **카드 Lua는 실행하지 않는다** — 의미를 번역한다. 못 옮기면 실행 말고 보존.
   (2026-07-15 실측으로 확정: 용사여관 Lua는 금지 API를 요구 — 태그 번역 경로가 옳다)
3. **보안 척추 불가침** — integrity 서명, BUTTON_ONLY_EVENTS, maskSecrets, 수치 발명 금지
   (회수/합성 구분을 진단·문서에 기록).
4. **각 슬라이스는 관통 단면** — 엔진→UI까지 세로로 뚫고, 매번 독립 감사.
5. **RNG 규율** — 굴림 횟수는 상태의 결정론 함수. 거부는 굴림 전, 문턱 미달도 소비.

## 리스크

- **LLM 태그 준수율**: 영수증에 "태그 없음"이 보이므로 감지 가능, 재생성으로 복구.
- **CBS/regex 범위**: 전체 재현은 늪 — 카나리아가 실제로 쓰는 부분집합만, 여권에 정직하게 표기.
- **카드 생태계**: 카드 원본은 사용자 로컬에만. 변환 결과 재배포·재수출 기능은 만들지 않는다.
- **시스템 인플레**: 묶음마다 수치 상한(명중 +5, 군수 ≤40%, 근무 ≤50%)을 걸고 통합 밸런스 패스에서
  재검증한다.

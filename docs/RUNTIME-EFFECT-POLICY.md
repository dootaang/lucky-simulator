# 카드 트리거 효과 처분표

실행 정책의 단일 원본은 `packages/risu/src/security/effect-policy.ts`다. 이 문서는 사람이 읽기 위한 설명이며, 테스트가 업스트림 `port/triggers.ts`에 새 효과가 추가됐는데 정책에 빠진 경우 실패한다.

| 처분 | 의미 | 대표 효과 |
|---|---|---|
| 적용 | Worker 안의 복사본에서 계산하고, 변수 patch로만 반환 | `setvar`, 조건·반복·문자열·배열·사전 계산, 읽기 전용 조회, `runtrigger` |
| 차단 | 실행하지 않고 `runtime_effect_blocked` 진단을 남김 | Lua/코드, 명령 실행, LLM·이미지·유사도 요청, 콘솔 출력 |
| 사용자 승인 필요 | 실행하지 않고 승인 요청 효과와 `runtime_effect_approval` 진단을 반환 | 채팅 삭제·수정, 프롬프트/대리발화, 알림 입력, 로어북·캐릭터·페르소나·글로벌 노트 변경 |

목록에 없는 새 효과는 기본 차단한다. 차단·승인 효과는 업스트림 실행기에 형태를 보존한 무해 효과로 전달하므로, 같은 트리거 뒤의 안전한 계산은 계속 실행된다. 현재 프로덕션 트리거 배선은 꺼져 있고 승인 UI도 만들지 않았다.

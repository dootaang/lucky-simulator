# 럭키★시뮬레이터 UI·UX 리뉴얼 마스터 설계

- 상태: 구현 전 기준본
- 작성일: 2026-07-20
- 대상: 모바일 우선 플레이어 셸, 데스크톱 적응형 셸, 설정, 온보딩, 관리 화면
- 우선 문서: [ADR 0001](adr/0001-product-identity-and-platform-boundaries.md), [DESIGN](DESIGN.md)
- 구현 여부: 이 문서는 화면과 사용자 흐름의 기준만 정한다. 엔진·저장·리플레이 로직은 변경하지 않는다.

## 1. 오너 브리핑

리뉴얼이 끝나면 처음 온 사용자는 다음 세 가지만 이해하면 된다.

1. 봇을 가져온다.
2. AI가 필요한 순간 연결한다.
3. `봇 · 대화 · 관리 · 설정` 네 메뉴로 모든 기능을 찾는다.

지금처럼 봇 목록과 채팅 양쪽에서 같은 전체 설정으로 들어가거나, 플레이 화면에서 엔진 컴파일·호환성 여권·세션 검사기를 동시에 마주치지 않는다. 고급 기능은 삭제하지 않고 `고급 도구` 안에 둔다. 한 번 고급 도구를 표시한 사용자는 다음 방문에서도 계속 볼 수 있다.

이 작업은 색을 새로 칠하는 스킨 교체가 아니다. 사용자가 기능의 소속을 먼저 추리해야 하는 현재 구조를 고치는 정보 구조 리뉴얼이다.

## 2. 제품 목표

### 2.1 핵심 목표

- 비코더가 설명 없이 3분 안에 카드 가져오기와 첫 플레이 화면 도달
- 모바일에서 주요 목적지를 한 번의 탭으로 이동
- 전체 앱, 현재 봇, 현재 대화에 적용되는 설정을 사용자가 구분
- 일상 플레이에는 내부 기술 용어를 노출하지 않음
- 소녀전선뿐 아니라 판타지봇·Alternate Hunters V2 등 후속 장르에도 같은 셸 사용
- 접근성, 가독성, 터치 편의성, 오류 복구를 공통 계약으로 고정

### 2.2 이후 목표

- 실사용자 측정 후 첫 성공 시간을 60초에 가깝게 단축
- 장르별 콘솔이 달라도 관리 탭의 첫 화면은 같은 요약 문법 유지
- 디자인 토큰과 공용 부품을 우회하는 새 코드가 자동 검사에서 실패

### 2.3 하지 않는 것

- 사용자용 `Easy/Pro` 두 모드를 만들지 않는다.
- 전체 관리 콘솔을 작은 바텀시트에 넣지 않는다.
- RisuAI나 SillyTavern UI를 통째로 복제하지 않는다.
- 엔진 사실, RNG, 저널, 저장 계약을 UI 편의를 이유로 변경하지 않는다.
- 숫자 근거가 불명확한 UX 성과를 제품 약속으로 사용하지 않는다.
- 화면 지도 없이 CSS부터 교체하지 않는다.

## 3. 현재 문제의 구조적 진단

### 3.1 같은 전체 설정의 중복 입구

현재 전체 설정은 다음 위치에서 열린다.

- 모바일 봇 목록 우측 톱니바퀴
- 모바일 현재 봇 메뉴를 연 뒤 우측 톱니바퀴
- 데스크톱 카드 스트립 상단 톱니바퀴

기능은 같지만 위치가 여러 개라 사용자는 서로 다른 설정인지 판단해야 한다. 새 셸에서는 전체 설정을 최상위 `설정` 목적지 하나로 통일한다.

### 3.2 서로 다른 적용 범위의 혼합

현재 설정 화면에는 전역 AI 모델, 현재 회차의 프리셋 토글, 페르소나, 채팅 표시, 진단, 저장 복구가 같은 층에 있다. 현재 봇 메뉴에는 채팅 목록, 봇 편집, 에셋, 엔진 준비, 시뮬레이션, 장기기억, 진단, 되돌리기가 함께 있다.

새 구조는 기능을 기술 분야가 아니라 사용자가 이해하는 적용 범위로 나눈다.

### 3.3 플레이어와 정비실의 혼합

다음 용어와 기능은 개발자에게는 유용하지만 첫 플레이에 필요하지 않다.

- 엔진 컴파일
- 호환성 여권
- 세션 검사기
- 진단 콘솔
- 프롬프트 블록 편집
- 서버 주소와 세부 생성 파라미터

이 기능은 `고급 도구`로 이동한다. 오류가 발생했을 때는 내부 도구 이름보다 사용자가 할 수 있는 해결 행동을 먼저 보여준다.

### 3.4 시각 규칙 우회

2026-07-20 플레이어 Svelte 소스의 단순 정적 계측 결과는 다음과 같다.

- 직접 작성된 HEX 색상: 946회
- 직접 작성된 RGB 색상: 9회
- 직접 작성된 크기·간격 선언: 약 804회
- 공통 색상·간격·반경 토큰 참조: 81회

`packages/ui/src/tokens.css`에 공통 규칙이 있지만 플레이어 화면 대부분이 이를 우회한다. 또한 플레이어가 사용하는 일부 의미 변수는 정식 토큰에 없고 장르 전용 토큰 계약도 없다.

처방은 기존 토큰을 버리는 것이 아니라 다음 순서다.

1. 기존 토큰의 이름과 의미를 정리한다.
2. 누락된 표면·보조문·상태·장르 토큰을 보충한다.
3. 공용 버튼·입력·시트·탭 부품을 완성한다.
4. 화면을 공용 규칙으로 점진 이주한다.
5. 직접 색상·간격 추가를 막는 정적 검사를 둔다.

## 4. 새 정보 구조

### 4.1 모바일 최상위 목적지

```text
┌────────┬────────┬────────┬────────┐
│   봇   │  대화  │  관리  │  설정  │
└────────┴────────┴────────┴────────┘
```

### 봇

- 봇 셸프를 홈으로 사용
- 검색, 정렬, 카드 가져오기
- 최근 사용 봇과 복구가 필요한 봇 표시
- 봇을 선택하면 해당 봇의 최근 대화를 이어감
- 삭제·원본 재연결 등 위험 행동은 봇 상세의 별도 메뉴로 이동

### 대화

- 현재 대화와 메시지 입력
- 응답 재생성·편집·번역 등 메시지 기능
- 현재 봇의 대화 목록
- 현재 회차에만 적용되는 빠른 설정
- AI 미연결 시 작성한 메시지를 보존한 채 연결 안내

### 관리

- 장르 중립적인 최상위 목적지
- 소녀전선에서는 지휘 콘솔
- 판타지봇에서는 상태·퀘스트·인벤토리·전투 관리
- 첫 화면은 장르별 세부 탭이 아니라 `지금 상태 · 지금 할 일 · 문제 · 최근 보상` 요약
- 전체 콘솔은 모바일 전체 화면 사용

### 설정

- 전체 설정의 유일한 진입로
- 앱 전체·현재 봇·현재 대화의 적용 범위를 구분
- 데이터·백업·도움말과 고급 도구 포함

### 4.2 데스크톱 적응형 구조

모바일의 네 목적지와 이름·순서를 유지한 채 왼쪽 세로 레일로 바꾼다.

```text
┌──────────┬──────────────────────┬─────────────────┐
│ 봇       │                      │ 선택적 보조 패널 │
│ 대화     │      주 작업 화면    │                 │
│ 관리     │                      │                 │
│ 설정     │                      │                 │
└──────────┴──────────────────────┴─────────────────┘
```

- 넓은 화면에서는 목록과 상세를 동시에 볼 수 있다.
- 모바일과 데스크톱에서 같은 기능의 이름과 상대적 순서를 바꾸지 않는다.
- 보조 패널은 사용자가 연 경우에만 유지하며 주 작업을 가리지 않는다.

## 5. 기능 적용 범위 계약

모든 설정 항목은 아래 범위 중 하나를 선언해야 한다. 화면에는 범위를 사람말로 표시한다.

| 범위 | 사용자 표시 | 대표 기능 |
|---|---|---|
| 앱 전체 | 모든 봇에 적용 | AI 서비스, 기본 모델, 화면, 접근성, 언어, 저장·백업 |
| 현재 봇 | 이 봇만 | 에셋 모듈, 봇 정보, 봇 고정 페르소나, 게임 기능 준비, 봇 편집 |
| 현재 대화 | 이 대화만 | 프리셋 토글, 장기기억, 회차 설정, 대화 내보내기·초기화 |
| 고급 도구 | 문제 해결·고급 | 프롬프트 블록, 세부 파라미터, 가져오기 진단, 세션 검사기, 기술 진단 |

### 5.1 범위가 아직 불명확한 기능

다음 항목은 구현 전 저장 계약을 확인하고 범위를 확정한다.

- 활성 프롬프트 프리셋 자체의 전역/봇별 적용
- 페르소나 기본값과 봇 고정값의 우선순위
- 채팅 이미지 크기의 기기별/계정별 적용
- 고급 도구 표시 상태의 저장 위치

기준안은 `고급 도구 표시 여부`만 기기 전체에 기억하고, 개별 접힘 항목의 상태는 영구 저장하지 않는 것이다.

## 6. 첫 실행과 AI 연결

### 6.1 첫 실행 흐름

```text
앱 진입
  → 봇카드 가져오기
  → 카드 분석
      ├─ 엔진 준비 완료: 인사말 + 이미지 + 로컬 게임 체험
      ├─ 일반 대화 카드: 인사말 + 이미지, 첫 전송 때 AI 연결
      └─ 엔진 변환 필요: 인사말 + "게임 기능 준비에는 AI 연결 필요"
  → 첫 성공
```

카드 분석 중에는 `컴파일` 같은 내부 용어 대신 실제 진행 단계를 표시한다.

- 카드를 읽는 중
- 이미지 확인 중
- 대화를 준비하는 중
- 게임 기능을 준비하는 중

### 6.2 AI 연결 시트

AI 연결은 첫 화면을 막지 않고 실제 AI 응답이 필요한 순간에 연다.

```text
AI 서비스 선택
  → API 키 입력
  → 추천 모델 선택
  → 연결 확인
  → 성공 시 원래 행동 계속
```

필수 계약:

- 사용자가 작성한 첫 메시지를 연결 과정에서 잃지 않는다.
- 키 형식은 서비스 추천 힌트로만 사용하고 자동 확정하지 않는다.
- 실제 API 요청으로 연결 성공을 확인한다.
- 실패하면 기술 코드보다 해결 행동을 먼저 안내한다.
- 키 저장 위치와 전송 범위는 실제 구현을 확인한 뒤에만 문구로 약속한다.
- 서버 주소와 세부 파라미터는 고급 도구에 둔다.

### 6.3 기본 목적 프리셋

세부 파라미터 위에 쉬운 선택지를 제공하되 원래 조절 기능은 보존한다.

- 균형
- 풍부한 서사
- 빠른 응답
- 직접 조절

각 AI 서비스에서 실제로 지원되는 파라미터만 바꾸며, 프리셋별 매핑은 테스트로 고정한다.

## 7. 주요 화면 저해상도 와이어프레임

### 7.1 봇 셸프

```text
┌────────────────────────────┐
│ 럭키★시뮬레이터       검색 │
├────────────────────────────┤
│ [큰 봇카드 가져오기 영역]  │
│                            │
│ 최근 봇                    │
│ [이미지] 소녀전선  이어하기│
│ [이미지] 판타지봇 이어하기 │
│                            │
├────────────────────────────┤
│   봇   대화   관리   설정  │
└────────────────────────────┘
```

빈 상태에는 설명문보다 하나의 큰 가져오기 행동을 우선한다. 선택적으로 자체 제작 초경량 체험용 봇을 제공할 수 있으나 첫 리뉴얼의 선행 조건은 아니다.

### 7.2 대화

```text
┌────────────────────────────┐
│ ← 봇 이름        현재 봇 ⋯ │
├────────────────────────────┤
│                            │
│          대화              │
│                            │
├────────────────────────────┤
│ [빠른 행동 또는 상태 1줄]  │
│ [메시지 입력          전송] │
├────────────────────────────┤
│   봇   대화   관리   설정  │
└────────────────────────────┘
```

우측 `현재 봇 ⋯`에는 현재 봇·현재 대화 기능만 둔다. 전체 설정으로 가는 톱니바퀴는 넣지 않는다.

### 7.3 관리

```text
┌────────────────────────────┐
│ 관리 · 현재 봇             │
├────────────────────────────┤
│ 지금 상태                  │
│ [자원] [시간] [진행]       │
│                            │
│ 지금 할 일                 │
│ [수복 필요] [임무 보상]    │
│                            │
│ 최근 결과                  │
│ [획득·소모 요약]           │
│                            │
│ [전체 관리 기능 보기]      │
├────────────────────────────┤
│   봇   대화   관리   설정  │
└────────────────────────────┘
```

채팅 위에는 빠른 관리 요약을 바텀시트로 제공할 수 있다. 전체 콘솔은 관리 탭의 전체 화면에서 연다.

### 7.4 설정

```text
┌────────────────────────────┐
│ 설정                       │
├────────────────────────────┤
│ 앱 전체                    │
│ AI 연결                 >  │
│ 화면·접근성             >  │
│ 데이터·백업             >  │
│                            │
│ 현재 봇                    │
│ 이미지·봇 정보          >  │
│                            │
│ 현재 대화                  │
│ 분위기·기능             >  │
│ 장기기억                 >  │
│                            │
│ [고급 도구 표시]           │
├────────────────────────────┤
│   봇   대화   관리   설정  │
└────────────────────────────┘
```

|---|---|---|
| Provider | AI 서비스 | 프로바이더 |
| Model | AI 모델 | 호출명 |
| Engine compile | 게임 기능 준비 | 엔진 컴파일 |

| Session inspector | 진행 기록 | 세션 검사기 |
| Diagnostics | 문제 해결 | 진단 콘솔 |
| Endpoint | 서버 주소 | 엔드포인트 |
| Ledger | 기록 | 원장 |
| Receipt | 결과표 | 영수증 |

규칙:

- 버튼은 내부 명사가 아니라 사용자가 일으킬 결과를 쓴다.
- 설명이 필요한 용어는 먼저 이름을 쉽게 바꾼다.
- 여전히 설명이 필요한 소수 항목만 `이게 뭔가요?` 펼침 설명을 사용한다.
- 모바일에서 호버 전용 툴팁을 만들지 않는다.

### 9.3 접근성 기준

- 모바일 터치 영역 최소 44×44 CSS px, 가능하면 48×48
- 주요 버튼은 아이콘과 글자를 함께 표시
- 아이콘만 사용할 경우 접근 가능한 이름 제공
- 키보드 포커스를 명확히 표시
- 고정 헤더·하단 바가 포커스와 입력창을 가리지 않음
- 색상만으로 상태를 구분하지 않음
- 본문 최소 13px를 하한으로 하고 핵심 조작은 더 크게 표시
- 애니메이션 감소 설정을 존중
- 모든 진행·성공·실패 상태를 보조기술이 읽을 수 있도록 알림

### 9.4 자동 검사

다음 규칙을 정적 검사 또는 린트로 고정한다.

- 플레이어 컴포넌트에 신규 직접 HEX/RGB 색상 금지
- 승인되지 않은 px 간격·반경·글자 크기 추가 금지
- 공용 버튼과 입력을 우회하는 신규 로컬 스타일 금지
- 아이콘 전용 버튼의 접근 가능한 이름 누락 금지
- 동일 기능의 서로 다른 사용자 명칭 금지

점진 이주 중에는 기존 위반을 기준 목록으로 고정하고 신규 위반부터 차단한 뒤, 파동별로 기준 목록을 줄인다.

## 10. 위험 행동과 오류 복구

- 삭제·초기화·연결 해제는 일상 행동과 시각적으로 분리
- 위험 행동은 화면 최하단 `위험 구역`에 둠
- 오류에는 `무엇이 실패했는지 → 사용자가 겪는 영향 → 지금 할 행동` 순서로 표시
- API 연결 실패는 키·모델·네트워크·서비스 주소를 구분해 안내
- 카드나 에셋 원본 소실은 복구 가능 범위에 맞는 안내 사용
- 사용자가 입력한 메시지·설정 초안은 오류 시 보존
- 진행 중인 버튼은 즉시 눌림 상태를 보이고 중복 실행을 막음

## 11. 사용자 시험

에이전트 합의는 설계 가설일 뿐 실제 사용자 증거가 아니다. 베타 커뮤니티의 비코더 사용자를 중심으로 과업 시험을 한다.

### 11.1 필수 과업

1. 빈 앱에서 봇카드 가져오기
2. AI 연결 후 첫 메시지 보내기
3. 기존 봇의 최근 대화 이어가기
4. 관리 화면에서 지금 할 일 찾기
5. 대화 분위기·기능 또는 이미지 크기 바꾸기
6. 장기기억 확인하기
7. 채팅 백업하기
8. 잘못 연결된 AI 설정 문제 해결하기

### 11.2 기록 항목

- 과업 성공 여부
- 완료 시간
- 첫 클릭 위치
- 잘못 누른 횟수
- 도움 요청 여부
- 사용자가 소리 내어 말한 혼란 지점
- “이 설정이 어디에 적용된다고 생각하는가”에 대한 답

화면 녹화와 로그에는 실제 API 키가 노출되지 않게 테스트 키·가림·중단 절차를 둔다.

### 11.3 1차 합격선

- 신규 비코더가 설명 없이 3분 안에 첫 플레이 화면 도달
- 필수 과업 1~5의 성공률 80% 이상
- 전체 설정 진입 위치를 한 번에 찾음
- 앱 전체·현재 봇·현재 대화 범위를 잘못 이해하는 참가자가 20% 미만
- 모바일 핵심 과업 중 가려진 버튼·입력창 0건
- 기존 파워유저가 고급 기능을 잃지 않음

참가자 수가 적을 때 비율을 제품 성과처럼 과장하지 않고, 반복되는 실패 유형을 수정 우선순위로 사용한다.

## 12. 구현 파동

### 파동 0 — 메뉴 감사와 시제품

- 플레이어 컴포넌트의 기능과 실제 진입로 전수 조사
- 각 기능을 네 적용 범위로 분류
- 저장 계약이 불명확한 항목 확인
- 모바일 저해상도 와이어프레임과 클릭 시제품 제작
- 비코더 과업 시험

완료 조건: 코드를 옮기기 전에 새 화면에서 모든 기존 기능의 목적지가 정해져 있다.

### 파동 1 — 디자인 기반

- 기존 토큰 정리와 누락 토큰 확장
- 공용 버튼·필드·시트·탭·상태 부품
- 신규 우회 방지 검사
- 기존 화면의 시각은 아직 대규모 변경하지 않음

완료 조건: 이후 새 셸이 로컬 HEX와 임의 간격 없이 구현 가능하다.

### 파동 2 — 적응형 셸과 4탭

- 모바일 하단 4탭
- 데스크톱 동일 순서 내비게이션 레일
- 설정 진입로 일원화
- 기존 화면을 새 목적지에 연결하되 기능 삭제 없음
- 하단 바·키보드·안전 영역 충돌 해결

완료 조건: 봇·대화·관리·설정을 한 번의 조작으로 이동하고 중복 전체 설정 입구가 없다.

### 파동 3 — 첫 실행과 AI 연결

- 카드 가져오기 중심 빈 상태
- 카드 상태 3분기
- 지연형 AI 연결 시트
- 연결 시험과 메시지 초안 보존
- 쉬운 목적 프리셋

완료 조건: 키가 필요한 시점까지 사용자가 카드의 가치와 다음 행동을 이해한다.

### 파동 4 — 설정 범위 재편

- 앱 전체·현재 봇·현재 대화 그룹
- 각 항목에 적용 범위 표시
- 고급 도구 표시 상태 기억
- 쉬운 용어와 보조 기술명
- 위험 구역 분리

완료 조건: 사용자 시험에서 설정 범위 혼동이 합격선 아래다.

### 파동 5 — 관리 요약층

- 장르 중립 관리 요약 계약
- 소녀전선 요약 화면 적용
- 빠른 관리 바텀시트
- 전체 콘솔은 전체 화면 유지
- 후속 장르가 같은 요약 계약을 사용하도록 문서화

완료 조건: 세부 탭을 탐색하지 않아도 현재 상태와 다음 할 일을 파악한다.

### 파동 6 — 화면별 시각 이주

- 사용 빈도가 높은 화면부터 공용 디자인 체계로 이주
- 직접 색상·크기 기준 목록 감소
- 접근성·반응형·가독성 회귀 검사
- 불필요한 중첩 프레임과 작은 글씨 제거

완료 조건: 신규 우회 0건, 대상 화면의 기존 우회 0건.

### 파동 7 — 통합 사용자 검증과 정리

- 모바일·태블릿·데스크톱 과업 시험
- 비코더와 기존 파워유저 모두 검증
- 중복 임시 셸과 롤아웃 플래그 제거
- 도움말·백업 안내·오류 문구 확정

완료 조건: 1차 합격선과 엔진·리플레이 회귀 검사를 모두 통과한다.

## 13. 테스트 계약

UI 리뉴얼은 다음을 동시에 지켜야 한다.

- 기존 세션과 저장을 그대로 열 수 있음
- 동일 시드·동일 행동의 엔진 결과와 저널이 동일
- 메시지 작성 중 설정·연결 시트를 열어도 초안 보존
- 모바일 뒤로가기·Escape·포커스 복귀가 예측 가능
- 네 목적지의 이름·순서가 화면 크기에 따라 바뀌지 않음
- 관리 화면에서 장르별 기능 손실 없음
- 설정 저장 범위가 화면 표기와 일치
- 스크린리더 이름, 키보드 탐색, 터치 영역, 대비 검증

## 15. 잠긴 결정과 보류 결정

### 잠긴 결정

- 모바일 홈은 봇 셸프
- 최상위 목적지는 `봇 · 대화 · 관리 · 설정`
- 전체 설정 진입로는 설정 목적지 하나
- 설정은 앱 전체·현재 봇·현재 대화·고급 도구로 분류
- Easy/Pro 이중 모드 없음
- 고급 도구 표시 여부는 기억
- 전체 관리 콘솔은 모바일 전체 화면
- 빠른 관리만 바텀시트 허용
- 첫 성공 합격선은 우선 3분
- 기술 용어는 기본 화면에서 사람말로 번역

### 보류 결정

- 자체 제작 초경량 체험용 봇의 내장 여부
- 60초 첫 성공을 공식 목표로 승격할 시점
- 사용자 계정·동기화 도입 후 기기별 UI 설정의 동기화 범위

보류 결정은 파동 0의 사용자 시험이나 해당 기반 기능이 준비되기 전까지 구현을 막지 않는다.

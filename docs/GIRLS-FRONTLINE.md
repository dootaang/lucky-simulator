# 소녀전선:잔불 지원

## 플레이어에게 보이는 결과

`소녀전선_잔불.png`를 가져오면 Lucky가 카드 안의 구조를 알아보고 소녀전선 전용 지휘 콘솔을 바로 연다. 별도 AI 컴파일이나 Lua 실행 승인을 요구하지 않는다.

- 지휘관/전술인형 시작
- 보유 인형 0명으로 시작, 고용·제조 뒤 제대 편성
- 매일 5명 고용 목록, 하루 1명 계약, 지정 고용과 숙소 정원
- 레드·오렌지/동유럽/화동 전선을 먼저 고른 뒤 세부 지역 선택, 결정론 전투와 승리 보상
- 제조와 수복 대기열
- 기지 시설 강화와 MOD 개조
- 카리나 보급 상점/장비 상점/보관함·장비 관리의 단계형 진입
- 전술요정 계약·제대 배치
- 지휘관실·작전실·정비실·숙소·기지 외부 이동, 현장/원격 지휘 구분
- 모바일 세로 및 가로 화면

업데이트판의 `v0.5fix 소녀전선 에셋 모듈 1번.charx`부터 `3번.charx`까지 함께 선택하면 원본 파일 3개는 브라우저 저장소에 각각 보존하고 ZIP 목차만 먼저 읽는다. 필요한 인형·감정·배경 이미지만 실제 화면이 요청할 때 해당 모듈에서 꺼내므로 대용량 파일 전체를 한꺼번에 해제하거나 미리 읽지 않는다.

## 가져오는 방법

1. `카드 가져오기`를 누른다.
2. `소녀전선_잔불.png`와 에셋 모듈 1·2·3번을 한꺼번에 선택한다.
3. 세 모듈의 합계 용량이 표시된 대용량 연결 안내를 한 번 승인한다.
4. 자동으로 열린 `그리폰 지휘부`에서 시작 방식을 고른다.

에셋 모듈 없이 카드만 먼저 가져온 뒤, 에셋 탭의 `에셋 모듈 연결`에서 1·2·3번을 함께 골라 나중에 붙여도 된다. 모듈 수에는 1개 제한이 없고, 같은 모듈을 다시 선택해도 중복 연결되지 않는다.

## 호환 방식과 안전 경계

Lucky는 원본 카드의 20만 자 Lua를 그대로 실행하지 않는다. 카드에서 인형·장비·임무·시설·제조 규칙을 정적으로 읽어 검증된 `genre.gfl` 모듈과 공용 전투·시간·상점 모듈로 다시 구성한다.

따라서 원본 Lua가 사용하던 채팅 수정, DOM 조작, 저수준 API 권한은 열리지 않는다. 자금·자원·전투 결과는 엔진이 먼저 확정하고 AI는 대화와 장면만 연출한다.

원본의 45개 표시 정규식과 고정 위치 HTML/CSS는 인증판에서 렌더하지 않는다. 모바일 화면을 덮고 버튼은 작동하지 않는 껍데기 대신 Lucky 네이티브 지휘 콘솔을 사용하며, 대화 화자가 `M4A1: ...`처럼 확인되면 해당 인형 기본/감정 에셋을 전술 프레임 안에 표시한다.

## 장면 BGM (8번)

옛 제작자판의 `GF_BGM.js`와 `Girls' Frontline_ Ember (1).charx`를 함께 조사했다. 카드 안에는 오디오 파일이 없으며, 원본도 카드나 에셋 모듈에서 음악을 읽지 않았다. AI 답변의 `BG_위치|...|BGM_day|` 또는 `|BGM_day|` 신호를 별도 Risu 플러그인이 읽어 외부 YouTube 음원을 반복 재생하는 구조였다.

Lucky 인증판은 원본 JavaScript를 실행하지 않고 다음 동작만 안전한 전용 기능으로 옮긴다.

- `bgmoff`와 30개 곡 신호를 허용 목록으로 해석하며 임의 URL·영상 ID는 거부
- `BG_...` 구형 헤더와 `|BGM_...|` 신형 단독 태그를 모두 인식
- 기계용 BGM 태그는 본문에서 숨기고, 채팅 상단의 작은 `SCENE BGM` 바에 현재 분위기를 표시
- 처음에는 외부 사이트에 연결하지 않으며 사용자가 재생을 누른 뒤에만 `youtube-nocookie.com` 플레이어 생성
- 재생/일시정지, 이전/다음, 음소거, 음량, 접기 지원; 모바일 세로·가로에서는 채팅과 입력창을 가리지 않음
- 사용자가 다음 말을 입력하는 동안에는 직전 장면 음악을 유지하고, 새 AI 답변의 신호가 오면 곡을 교체

2026-07-17 점검에서 원본 30개 영상 ID 모두 YouTube oEmbed 응답 200을 반환했다. 다만 외부 영상은 게시자가 삭제하거나 네트워크가 끊기면 재생되지 않을 수 있다. 이 경우 게임 상태와 대화는 영향을 받지 않으며 BGM 바만 소리가 나지 않는다. 장차 권리와 배포 용량이 정리된 로컬 음원 팩이 제공되면 같은 곡 신호에 연결할 수 있도록 큐 해석과 음원 공급을 분리했다.

옛 카드 실측은 JPEG 앞부분 뒤에 ZIP이 붙은 154.5MB CHARX, ZIP 항목 3,300개, 이미지 1,649개, 오디오 0개, 내장 `module.risum` 정규식 80개·트리거 155개다. BGM 재생 자체는 이 거대 모듈이 아니라 15.8KB 외부 플러그인이 담당했다.

이 정적 컴파일은 필요하지만 Lucky가 가져오기와 카드 열기 때 자동으로 수행한다. 모델 API 키나 사용자의 컴파일 승인은 필요 없다. 이미 저장된 소녀전선 카드도 최신 인증 템플릿을 우선 적용하므로 구버전 초안에 머물지 않는다.

## 원본 고용·시설 규칙

원본 카드의 고용 규칙을 그대로 네이티브 상태로 옮긴다.

- 시작 보유 인형은 0명이며 M4A1을 자동 지급하지 않음
- 미보유 인형 중 5명을 매일 추첨
- 같은 날 계약은 한 명만 가능
- 일반 계약은 카드의 `<인형명>d` 변수에 기록된 개별 가격 사용
- 저격 고용은 개별 가격에 자금 3,000 추가
- 숙소 레벨별 정원 3 / 6 / 9 / 12 / 15명
- 계약 직후 상태는 `이동 중`, 다음 시간대에 `대기`로 전환

시설은 Lv.1에서 시작해 Lv.5가 최대이며 다음 비용은 직전 비용의 1.5배를 내림한다.

| 시설 | Lv.1→2 | Lv.2→3 | Lv.3→4 | Lv.4→5 |
| --- | ---: | ---: | ---: | ---: |
| 훈련 시설 | 4,000 / 2,000 | 6,000 / 3,000 | 9,000 / 4,500 | 13,500 / 6,750 |
| 방위 시설 | 4,000 / 2,000 | 6,000 / 3,000 | 9,000 / 4,500 | 13,500 / 6,750 |
| 정비 시설 | 4,000 / 2,000 | 6,000 / 3,000 | 9,000 / 4,500 | 13,500 / 6,750 |
| 보급소 | 5,000 / 3,000 | 7,500 / 4,500 | 11,250 / 6,750 | 16,875 / 10,125 |
| 인형 숙소 | 3,000 / 1,000 | 4,500 / 1,500 | 6,750 / 2,250 | 10,125 / 3,375 |

표의 앞 숫자는 자금, 뒤 숫자는 자원이다.

## 회수와 합성의 구분 (2026-07-17)

수치는 발명하지 않는 것이 원칙이다. 실카드 기준:

- **원본에서 회수**: 인형 206/211명의 HP·MP·전투력·기분(카드 `defaultVariables`의 인형별 배열),
  병과·등급, 아이템·장비·임무·시설 기본 비용 테이블, MOD 개조 전투력(MOD_POWER), 지휘관 시작 자금 10,000.
  카드에 배열이 없는 5명은 원본 Lua 자신의 폴백(HP 1000 · MP 1000 · 전투력 500 · 기분 50)을 쓴다.
- **Lucky 규칙으로 합성**: 전투 판정(d20·치명타·전리품 범위), 제조 비용, 시설 강화 레벨 배율.
  합성 목록은 컴파일 결과 `issues`에 남고 카드 열기 시 진단 콘솔에 기록된다 — 조용한 확정은 없다.

콘텐츠 권리·공유 경계는 [CONTENT-POLICY.md](CONTENT-POLICY.md)를 따른다.

## 실카드 카나리아

다음 명령은 실제 카드와 에셋 모듈을 읽어 자동 탐지, 네이티브 시작·편성, 대용량 ZIP 지연 인덱싱을 한 번에 검사한다.

```powershell
corepack pnpm --filter @simbot/web canary:gfl -- "C:\freetalk\소녀전선\업데이트버전\소녀전선_잔불.png" "C:\freetalk\소녀전선\업데이트버전\v0.5fix 소녀전선 에셋 모듈 1번.charx" "C:\freetalk\소녀전선\업데이트버전\v0.5fix 소녀전선 에셋 모듈 2번.charx" "C:\freetalk\소녀전선\업데이트버전\v0.5fix 소녀전선 에셋 모듈 3번.charx"
```

2026-07-17 기준 실측:

- 카드 내장 에셋 146개
- Lua 235,589자, 초기 변수 2,463줄, 정규식 45개
- 네이티브 변환 인형 291명, 임무 48개
- M4A1 계약가 7,000, 일일 고용 후보 5명, Lv.1 숙소 정원 3명
- 훈련 시설 1·2차 증설비 4,000/2,000 및 6,000/3,000
- 에셋 모듈 3개, 총 41,932개 항목 중 이미지 20,963개
- ZIP 중앙 목차 합계 3,341,069바이트, 데스크톱 Node 3개 순차 인덱싱 37ms

모바일의 실제 시간과 저장 용량은 기기·브라우저마다 달라질 수 있으므로 대용량 연결 전 안내를 표시한다.

**모바일 실증(G1): 2026-07-17 오너 실기기 스모크 통과** — 이전판 카드+554MB 단일 에셋 모듈 임포트, 인형·배경 표시,
지휘 콘솔 전 탭 동작 확인. 업데이트판 3개 모듈은 데스크톱 실파일 인덱싱과 브라우저 다중 연결 E2E를 통과했으며, 모바일 총 1.18GB 저장은 기기별 여유 용량 확인이 필요하다.

# 결과 보고 — GFL 관계 티어 정합 핫픽스 (2026-07-18)

## 결과

실카드의 음수 관계 단계가 이제 실제 플레이에 반영된다. 적대·경계·불편 상태에서는 상대를 달랠 수 있는
`차분히 대화한다`만 보이고, 호감도 0부터 별명·격려가 열린 뒤 20/50/80/150 문턱마다 깊은 교류가 순서대로
해금된다. 호감도 400은 `???`로 건너뛰지 않고 `서약`으로 표시된다.

## 구현과 해석

- `RelationChoice.minTier`를 중립 티어 기준 `tierOffset`으로 바꿨다. 현재 티어가
  `호감도 0의 티어 + 오프셋` 이상일 때만 선택지를 허용하며, 카드 티어 수를 넘는 요구치는 마지막 티어로
  접는다. 관계 설정이 없는 카드는 기존처럼 9종 모두 열린다.
- 판정 전 게이트, 선택지 표시, 성공 뒤 후속 캡슐 필터가 모두 같은 계산 함수를 사용한다.
- 연속한 같은 threshold는 첫 번째 항목만 도달 가능하게 해석한다. 원본 Lua와 동작이 다를 수 있지만
  `REL_NAMES`와 `REL_THRES` 회수 데이터는 삭제·수정하지 않고 해석만 고정했다.
- 기존 talk/nickname/encourage의 id·DC·증감 수치는 바꾸지 않았다.

## 검증

- `corepack pnpm check` 통과: 타입 검사, 단위 테스트 319건(모듈 80·세션 120·웹 119), 프로덕션 빌드가 모두 성공했다.
- `corepack pnpm test:e2e` 통과: Chromium 실제 화면 동선 44건이 모두 성공해 관계 대화와 기존 플레이 화면이 함께 유지됨을 확인했다.
- `corepack pnpm --filter @simbot/web canary:gfl` 통과: 실카드 1건과 에셋 3개(항목 41,932개)를 읽어 인형 291명·작전 48개를 정상 복원했다.
- 관계 경계값 단위 테스트 통과: 호감도 `-100/0/25/55/85/150/400`에서 선택지 수가 각각 `1/3/5/7/8/9/9`로 열리고, 400점 표시는 `서약`으로 고정된다.

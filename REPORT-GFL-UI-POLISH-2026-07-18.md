# GFL UI 폴리시 완료 보고

## 결과

인형 수백 명을 긴 선택 상자에서 찾던 화면을 검색 가능한 카드 목록으로 바꿨다. 관리 행동 뒤에도 창을 유지하도록 선택할 수 있고, 창을 유지한 상태에서는 영입·대화·작전 후속 결정을 관리창 아래에서 바로 처리한다. 기지 시설과 작전 보고는 처음에는 핵심 정보만 보여 주고 필요한 항목을 펼쳐 확인할 수 있다.

엔진 사건·수치·RNG는 변경하지 않았다. 모듈에는 화면이 사용할 `gfl/formation/guide` 셀렉터 한 개만 추가했다.

## 인형 선택기 적용처

하나의 `DollPicker.svelte`를 다음 세 곳에서 재사용한다.

| 적용처 | 화면 동작 | DOM 검증 근거 |
|---|---|---|
| 저격 고용 | 이름 검색 후 가격이 표시된 후보를 지정 계약 | `전술인형 선택기` 안 `인형 이름 검색`에 `M4A1` 입력 후 `.results article` 1개, `지정 계약` 클릭 후 숙소 `1/4` |
| 제대 배치 | 검색·필터된 인형을 선택해 현재 슬롯에 배치 | 기존 6칸 제대 e2e가 `.roster button`으로 배치하고 SLOT 6까지 채움 |
| 인형 관리 | 같은 필터 바와 카드 그리드에서 상세 대상을 선택 | 해체·보스 영입 e2e가 `.doll-grid button` 개수와 선택 동작을 검증 |

검색은 NFKC 정규화와 대소문자 무시를 적용했다. 병과 칩은 입력 rows에 실제로 존재하는 병과만 만든다. 최소 ★, 이름·★·전투력·가격 정렬, 최초 50개와 `더 보기`를 제공한다. 초성 검색은 이번 범위에서 구현하지 않았다.

선택기 모델 단위 테스트 5건이 이름 검색, 병과 다중 필터, 최소 등급, 전투력·가격 정렬, 50개 더 보기를 각각 검증한다.

## 관리창

`DecisionDock` 자체를 `SimulationPanel` 하단에 재사용해 채팅 쪽 카드와 같은 모델·같은 행동 함수를 쓴다. 별도 결정 로직을 복제하지 않았다. e2e에서 1:1 대화를 시작한 뒤 관리창 내부의 `엔진 결정 카드`에서 `대화를 마무리한다`를 눌러 카드가 사라지는 것까지 검증했다.

설정의 기타 → 시뮬레이션에 `관리 행동 후 관리 화면 유지`를 추가했다. 기본값은 꺼짐이며 기존 `simbot.llm` localStorage 저장 경로에 함께 보존된다. 전투가 시작되면 관리창을 다시 여는 기존 자동 인터럽트는 그대로다.

## 작전·기지·모바일

- 임무 카드에 셀렉터 데이터 기반 세력 배지, 완료 ✓, 보스 👑을 표시한다.
- 전투 보고는 크리티컬과 ⚡ 상성 교전을 굵고 밝게, 빗나감을 흐리게 표시한다. 6라운드 뒤는 `라운드 더 보기`로 접는다.
- 시설 카드는 기본 상태에서 이름·Lv·증설 가능 여부만 보이고, 한 장을 누르면 현재/다음 효과·비용·증설 버튼이 열린다. 다른 카드를 열면 이전 카드는 닫힌다.
- 700px 이하 헤더는 `Lv·자금·자원·시간` 한 줄만 먼저 보이며 탭하면 전체 상태가 열린다.
- 390×844 e2e에서 콘솔의 `scrollWidth <= clientWidth + 1`을 확인했다.

## 검증

- GFL 브라우저 여정: 10/10 통과. 신규 3건(저격 검색 계약, 결정 카드 미러링, 시설 확장)과 기존 7건이 함께 통과했다.
- DollPicker 모델: 5/5 통과.
- GFL 모듈: 54/54 통과. 권장 열·세력 배지 셀렉터 계약을 포함한다.
- 전체 `pnpm check`: 타입 검사, 단위 테스트, 프로덕션 빌드 모두 통과했다. 웹 단위 테스트는 25파일 127건, GFL 모듈은 포함된 전체 모듈 테스트 12파일 101건이 통과했다.
- 전체 Playwright e2e: 49/49 통과했다. 기존 46건을 보존하고 신규 GFL UI 여정 3건이 추가됐다.

# REPORT — GFL ROMANCE-BALANCE (2026-07-19, 지휘자 직접 구현)

지시서: WORKORDER-GFL-ROMANCE-BALANCE-2026-07-19.md
오너 문제 제기: "대화로 호감도 쌓기는 너무 어려운데 선물주면 너무 쉬움. 선물 너프 없이 대화 쪽에서 해결."

## 구현 요약

### A. 승급 게이트 — 수치와 확정 티어의 분리 (핵심)
- `unit.confirmedTier`(확정 티어)를 신설. 표시·해금·게이트는 전부 `effectiveRelation = min(수치 티어, 확정 티어)`.
- 선물은 수치를 자유롭게 올리지만 티어 **확정**은 함께한 시간으로만:
  - 대화 성공 = 한 단계씩, **중립+3(신뢰)** 까지
  - 1:1 세션 마무리·외출 = 한 단계씩, **중립+5(사랑)** 까지
  - **서약**은 반지 의식(`item/use` oath 분기)으로만
- 하락은 게이트 없음 — 수치가 떨어지면 확정도 즉시 동기화(`syncTierDown`).
- 하위 호환: `confirmedTier` 없는 기존 저장은 첫 상호작용에서 **현재 값 티어로 초기화**(소급 강등 없음).
  `acquire()`는 중립 티어로 초기화. 봉인 마이그레이션은 스키마(문턱표)를 받지 못하므로
  정규화는 런타임 지연 초기화(`ensureConfirmedTier`)가 담당 — 멱등이며 이벤트 전(pre-delta) 스냅샷 기준.

### B. 대화 수치 리밸런스
- 성공 수치 2배: talk 6 / nickname 10 / encourage 16 / ask-past 10 / coffee 8 / train 14 / walk 12 / confide 20 / promise 24 / hobby 12
- 실패 위로 +1 (대실패 제외) — "서툴렀지만 노력은 전해졌다"
- 세션 마무리 보너스 +2 → **+5**
- 연속 성공 콤보: 판정 보정 +1~+3 (같은 인형 streak, 실패 시 리셋)

### C. 이야기꽃 — 선물과 대화의 시너지
- 선물(aff+) 직후 **같은 시간대**의 첫 대화 성공 1.5배. 1회 소비, 시간대가 지나면 소멸.

### D. 취향 곱연산
- 병과 선호 선택지 성공 시 1.5배 (기존 DC -2와 별도, 다른 배율과 곱연산).

### 금지사항 준수
- 선물 수치 무변경 ✓ / 서약 규칙·티어 문턱 회수값 무변경 ✓ / 하루 상한(4회) 무변경 ✓

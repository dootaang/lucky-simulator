# 결과 보고 — GFL 다단계 작전 + 임무 루팅 (2026-07-18)

## 결과

작전이 이제 한 번의 전투가 아니라 2~11개의 연속 단계로 진행된다. 출격 전에 정찰·소탕·섬멸을 고르고, 출격하는 순간 전체 단계가 봉인된다. 화면에서 단계 아이콘과 현재 위치를 확인하며 정찰·돌발·미확인 상황을 진행하고, 전투에서 잃은 HP와 얻은 전리품을 다음 단계까지 이어 간다. 중도 퇴각하면 전리품은 지키지만 완료 보상과 지휘 경험치는 받지 않는다.

## 회수와 합성

| 구분 | 내용 | 실카드 실측 |
| --- | --- | ---: |
| 원본 회수 | 별점별 단계 수 `PROG_BY_STAR` | 7개(3/5/7/8/9/10/11) |
| 원본 회수 | 임무 유형 이름·단계 보정·힌트 `MISSION_TYPES` | 3종 |
| 원본 회수 | 단계별 서사 지침 `EV_GUIDE` | 5종 |
| 원본 회수 | 아이템별 `drop%` | 19/19종 |
| 원본 사실 | 장비 드롭 | `EQUIP_DATA`에 drop 없음, 작전 루팅 제외 |
| Lucky 합성 | 단계 타입 가중치, 정찰 +1, 돌발 자원, mystery 발견률 | 엔진 데이터 표 |
| Lucky 합성 | 다단계 소모전 보정 | 피격량 −20%, 7단계 이상 교전 적 능력 −10% |

원본 제안 소탕 가중치 `battle 45 / recon 15 / other 15 / mystery 25`는 반복 교전이 매번 임무 전체 적 전력을 다시 내는 구조에서 4★ 완주율이 5.0%까지 내려갔다. 지시서의 ±20% 조정 범위 안에서 `25/20/25/30`으로 바꾸고, 단계 전투 피격량을 20% 낮췄으며 7단계 이상은 한 교전의 적 규모를 10% 낮췄다. 정찰·섬멸 가중치는 지시값을 유지했다.

## 결정론과 지급 규칙

- 시작 시 단계 수만큼 RNG를 정확히 한 번씩 소비해 전체 순서를 저장한다. 같은 시드는 같은 단계가 나오며 재개해도 다시 뽑지 않는다.
- battle 승리와 mystery 발견은 스키마 아이템 순서대로 모든 아이템을 각각 한 번 굴린다. 장비는 굴리지 않는다.
- 정찰 명중 +1은 다음 battle 한 번에만 적용하고 즉시 소모한다.
- 임무 보상·첫 클리어 처리·지휘 EXP는 최종 단계 승리 때 한 번만 지급한다. 단계별 HP와 전리품은 누적된다.
- 단계 로그의 원본 `guide`를 관리 서사화 시스템 프롬프트에 넣어, 정찰·돌발 단계에서 AI가 임의 교전을 만들지 못하게 했다.

# GFL 프롬프트 매크로 호환 완료 보고

## 플레이에서 달라지는 점

AI에게 전달되는 소녀전선 설정문에서 `{{getvar::A_day}}` 같은 상태 조회 표식이 그대로 노출되지 않고 실제 읽기 전용 값으로 바뀐다. 조건에 맞지 않는 설정 블록은 프롬프트에서 빠지고, 맞는 블록만 전달된다. 카드가 상태를 쓰거나 채팅·저장소를 조작하려는 매크로는 실행하지 않는다.

## 1단계 — 실카드 계수

`apps/web/scripts/macro-census.mts`로 `소녀전선_잔불.png`를 정적으로 읽었다. Lua와 트리거는 실행하지 않았다. 전체 원문에서는 매크로 1,626회, 이름 10종을 찾았다.

## 2단계 — 처리 분류

### 안전 구현

- 상태 조회: `getvar`, `getglobalvar`, 직접 변수, `array_element`
- 비교: `equal/is`, `not_equal/isnot`, `greater/greaterequal`, `less/lessequal`
- 숫자식: `calc`, `?` — 숫자와 산술·비교 기호만 받는 자체 계산기이며 JavaScript `eval`을 쓰지 않는다.
- 조건 블록: `#if`, `#if_pure`, `:else`, `/`, `/if`
- 표시 줄바꿈: `br`, `newline`

모두 입력 변수의 복사본을 읽기만 하며 난수·시계·저장소·채팅을 건드리지 않는다. 계약 버전은 `Prompt CBS read-only subset v0.2`로 코드에 명시했다.

### 정적 소거

실카드에서 실행 의미가 없다고 확실히 입증할 수 있는 별도 색·아이콘 매크로는 0종이었다. 수치를 맞추기 위한 임의 소거는 하지 않았다.

### 보존·경고 유지

- `setvar`: 상태 변경이므로 실행 금지.
- `source`: 사용자/에셋 소스 권한 조회라 프롬프트 계층에서 실행 금지.
- `M. STARNIER`, `캐릭터 명`: 카드 원문의 미정 변수·예시 자리표시자라 추측 치환 금지.
- 알 수 없는 매크로와 짝이 맞지 않는 조건 닫기는 원문에 남기고 경고한다.

같은 미지원 이름은 프롬프트 한 번에 한 건만 집계한다. 첫 원문과 위치는 보존하므로 경고를 숨기는 방식이 아니다.

## 전후 실측

| 항목 | 변경 전 | 변경 후 |
|---|---:|---:|
| 항상 활성 로어 미지원 경고 | 945 | 3 |
| 감소 | - | 942건, 99.68% |
| 합격선 | - | 90% 이상 — 통과 |

남은 이름 3종은 `/`, `M.`, `/if`다. `M. STARNIER`라는 미정 변수가 낀 비정상 조건문의 양쪽 닫기까지 함께 남은 결과이며, 값을 발명하지 않기 위해 보존했다.

## 검증

- 새 매크로별 골든 테스트를 포함한 전용 테스트 23건과 쓰기 금지·이름별 경고 중복 억제 검증을 추가했다.
- 용사여관과 Alternate Hunters의 비매크로 프롬프트를 UTF-8 바이트 배열로 비교해 변화가 없음을 확인했다.
- 기존 프롬프트 파리티는 미지원 표본을 `getvar`에서 금지된 `setvar`로 바꿔 “원문+경고 보존” 계약을 계속 검증한다.
- 계수 재실행 명령: `pnpm --filter @simbot/web exec node --experimental-transform-types scripts/macro-census.mts`
